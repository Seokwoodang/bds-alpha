# Handoff: 부동산알파 (bds-alpha) — 부동산 투자 플랫폼

## Overview
부동산알파는 데이터 기반 부동산 투자 플랫폼입니다. 사용자는 매물을 검색·비교하고, 지역별 실거래 시세 흐름과 예상 수익률을 분석하며, 투자 가이드 콘텐츠를 읽고, 관심 매물을 저장할 수 있습니다. 핀테크 톤(차분한 블루/네이비, 신뢰감, 정돈된 데이터 UI)을 지향합니다.

총 7개 화면(홈, 매물 목록, 매물 상세, 시세 분석, 투자 가이드, 지도 탐색, 마이페이지)과 가이드 리딩 오버레이로 구성됩니다.

## About the Design Files
이 번들의 파일(`부동산알파.dc.html`, `ListingCard.dc.html`)은 **HTML로 만든 디자인 레퍼런스**입니다. 의도한 외관과 동작을 보여주는 프로토타입이며, 그대로 복사해 운영에 쓰는 코드가 아닙니다.

작업의 목표는 이 HTML 디자인을 **대상 코드베이스의 기존 환경(React/Next.js, Vue 등)과 패턴·라이브러리에 맞게 재구현**하는 것입니다. 환경이 아직 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요. 사용자는 Next.js를 선호합니다 — App Router + TypeScript + 클라이언트 컴포넌트 기반 권장.

> 참고: 원본은 "Design Component"라는 커스텀 런타임(`<sc-for>`, `<sc-if>`, `<dc-import>`, `renderVals()`)으로 작성돼 있습니다. 이 문법/런타임은 재현 대상이 아닙니다 — 마크업의 시각적 결과와 동작 로직만 참고하고, 대상 프레임워크의 표준 방식(map/조건부 렌더링/컴포넌트 props)으로 옮기세요.

## Fidelity
**High-fidelity (hifi)**. 최종 색상·타이포그래피·간격·인터랙션이 확정된 픽셀 단위 목업입니다. 아래 디자인 토큰과 컴포넌트 스펙의 정확한 값(hex, px, weight)을 그대로 사용해 재현하세요. 단, 매물 사진·지도는 플레이스홀더이며 실제 이미지/지도 SDK로 교체해야 합니다.

## Interactions & Behavior
- **네비게이션**: 단일 페이지 내 화면 전환(SPA) — 헤더 nav, 카드/버튼 클릭으로 screen 상태 변경. Next.js 구현 시 실제 라우트(`/`, `/listings`, `/listings/[id]`, `/prices`, `/guides`, `/map`, `/mypage`)로 분리 권장.
- **검색**: 인풋 입력값으로 제목+지역+동+유형 부분일치 필터(실시간).
- **필터**: 거래유형/지역/매물유형 — "전체"면 통과, 아니면 일치. 조합 필터.
- **정렬**: 추천순(원본 순서)·가격높은순·가격낮은순(priceNum)·수익률순(roi).
- **관심 저장**: 하트 토글 → saved 배열에 id 추가/제거. 헤더 배지 수, 마이페이지, 카드/상세 하트가 전부 연동(전역 상태). 하트 클릭은 카드 클릭과 분리.
- **가이드 오버레이**: 열기/닫기, 배경 클릭으로 닫힘.
- **차트/지도**: 지역 선택 시 라인차트·KPI·막대 강조·지도 핀이 함께 갱신.
- **애니메이션**: 화면 진입 `fade`/`fadeUp` (~0.4s ease), 카드 hover translateY(-3px)+shadow, 막대 height transition 0.4s. 버튼 hover 색상 전환.
- **반응형**: 데스크탑/모바일 모두 대응. 그리드 auto-fill/auto-fit, flex-wrap, `clamp()` 폰트로 처리. 헤더 nav는 flex-wrap.

## State Management
전역(또는 상위) 상태 권장:
- `screen` (현재 화면) / `selId` (선택 매물 id)
- `dealType`, `region`, `ptype`, `sort`, `q` (목록 필터/검색/정렬)
- `saved: number[]` (관심 매물 id 배열 — 전역 공유, localStorage 영속 권장)
- `selRegion` (시세/지도 선택 지역)
